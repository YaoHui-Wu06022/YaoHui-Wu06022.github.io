hexo博客的搭建以及fluid主题的配置

本文最后更新于：2022年3月31日 晚上

详细过程具体参考https://zhuanlan.zhihu.com/p/102592286；本文主要记录过程中一些问题

搭建的过程就是三部曲

补充：一定用网易UU加速器，省时间还快

绑定GitHub

ssh密匙指令：

绑定过程中不要频繁使用科学上网工具，绑定过程中会跳出登录页面，及时输入用户名和密码避免超时报错

过程中没有进行仓库的绑定，以后需要了再考虑

安装node.js和Hexo

需要重新设置路径：

设置完之后需要到系统的环境变量中添加路径

  1. 新建一个变量名为“NODE_PATH”，值为“D:\nodejs\node_global\node_modules”
  2. 编辑用户变量里的Path，将相应npm的路径改为：”D:\nodejs\node_global”，原来在C盘
  3. 在完成hexo init后还需增加一个”D:\blog\blog\node_modules\hexo\bin“否则hexo 显示无命令

在 cmd 命令下执行 npm install webpack -g 在 “D:\nodejs\node_global\node_modules”会生成一个文件夹

在D盘创建Blog文件夹后，网上大多是直接Git打开下载hexo，然后直接hexo init但这样其实会报错显示没有hexo

解决方法：在Blog文件夹中用行控制台，输入**npx hexo init blog**

之后正常操作就行

设置fluid主题

主要参考还得看官方：开始使用 | Hexo Fluid 用户手册

个人觉得用npm下载主题的方法不如直接下载压缩包解压到相应位置方便，解压到 Hexo 博客目录中的 themes 目录，并将解压出的文件夹重命名为 fluid。

在博客目录下创建 _config.fluid.yml，将主题的 config.yml 内容复制进去。
